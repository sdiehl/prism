-- `without alloc <block>` lifts the block to a synthetic top-level function that
-- carries the zero-allocation certificate and takes the block's captured locals
-- as parameters; the block becomes a call to it. A local lambda would be a heap
-- closure (which allocates), so the region must become a named top-level
-- function, over which the existing whole-call-tree `fbip` check then runs.
