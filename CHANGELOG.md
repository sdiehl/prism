# Changelog

## 0.9.0

- Coeffects: a usage row, written postfix on a type. `!` says what a computation does; `@` says how a value may be used. Ten facts are reserved with one canonical table (`portable`, `local`, `noescape`, `once`, `many`, `unique`, `aliased`, `noalloc`, `linear`, `bounded_stack`); rows are sets with a canonical order that enters the content hash; an unknown fact is a hard error, so no program can establish a private meaning before a checker exists. The one checked fact this release is the allocation certificate: `@ noalloc` at the root of a return annotation, replacing `without alloc` and `\ alloc` outright, and the retired block form's job is done by hoisting the region into its own certified function.
- Lineage: every served artifact explains itself through one typed graph format, `prism-lineage-graph-v1`, with digest-derived node identities and four variants sharing one envelope, one renderer, one verifier, and one differ. Builds, recorded runs, docs generations, and the world resident's timelines are all the same shape: content-addressed inputs, capability observations, produced artifacts, verification edges.
- Run lineage: `prism run p.pr --record run.replay --lineage run.plineage -- args` records a run sidecar naming the source, Std, and package roots, compiler identity, argv, environment reads, file inputs and writes, stdout, and the replay trace it explains, with the trace's own file relation recorded so verification reads the graph rather than a filesystem convention.
- Provenance: every capability observation carries a canonical hash, and recording and replaying a run produce identical event hashes, asserted by test rather than claimed. Variable-length values commit content digests, so hostile input cannot forge an event boundary. A mismatched replay names the failing event index and the operation it expected.
- Lineage queries: `prism lineage why SIDECAR OUTPUT` walks an output back through the request, its inputs, the trace, and the compiler identity, and works after the source files are gone; `prism diff` takes two sidecars and reports preserved, moved, added, and removed digests by logical key with a nonzero exit when anything moved; `prism lineage verify` rehashes what can be rehashed, and `--replay` verifies by replaying rather than by trusting the sidecar's own numbers.
- Certificates: a passed verification can be persisted as a digest-named certificate over the sidecar it verified (`replay-verified`, `lineage-verified`), riding the store's existing certificate discipline; a certificate carrying an unrecognized claim is rejected rather than trusted.
- Docs: `prism docs` emits a manifest of what it documented, the docs variant of the lineage graph, naming the roots, compiler identity, generator format, every page digest, and every doctest output hash; `--verify-manifest` rejects a stale page or a drifted root. Regenerating under the same roots is byte-identical.
- Package world: `prism pkg check-world` grows per-package gates, examples through the compiler-owned runner, doctests, committed replay traces, docs manifests, and a public-API surface of definition hashes that, against a baseline report, names exactly which public definitions changed behavior, by digest and never by path. Gates that do not apply report as not-run rather than passing silently.
- Usage facts: `prism dump usage-summary` tabulates, per definition, the allocation certificate, `fip`/`fbip` discipline, per-parameter borrow mask, and checked effect row, scoped to the program's own definitions so an imported library's rows never appear. The same facts project as an aligned markdown table (`usage-summary-md`, stable under the repo formatter) and JSON (`usage-summary-json`); a package may commit the markdown as a golden, which check-world regenerates and reports as a drift-naming `usage` gate. The gate is report-only by default; `prism pkg accept-usage` writes or reseats the golden byte-stably, `--strict-usage` opts a CI lane into failing on drift (a missing summary is never fatal, missing means not opted in), and the JSON report carries the gate's evidence as `usage`, `usage_drift`, `usage_format`, and `usage_tier`, the whole-program lowering tier the summary is headed by. `prism dump captures` classifies every closure capture portable, nonportable, or unknown for a move across a suspend boundary.
- CLI: twenty-six top-level commands regroup to eleven plus four noun groups (`exec`, `lineage`, `pkg`, `store`), with no legacy aliases. `replay` has one mode again; `diff` is the single diff verb over sources or sidecars; the old spellings fail with the usual unknown-command error.
- PRISM WORLD: a dedicated browser resident, a shared cellular universe whose law is real Prism code hashed by the compiler and whose state hash is computed in Prism with the same blake3 that content-addresses Core. Same-origin tabs converge by independently recomputing the same timeline; forks and scrubbing reuse the determinism machinery; editing the law moves the law hash in every tab; the timeline exports as a lineage graph the CLI can explain and structurally verify. The reserved panels say reserved, and nothing on the page claims a proof that did not run.
- Compiler: the pipeline stages carry their discipline in types. `ElaboratedCore`/`LoweredCore` newtypes make wrong-stage routing a compile error, Core Lint is stage-total over the effect and runtime node families and publicly callable as `lint_core` for embedders linting hand-assembled Core, one `Front` runner derives every frontend consumer with the intentional divergences as named options, and one `BuildIdentity` computes the identity surface every consumer previously assembled piecewise.
- Step ruler: `prism exec steps` runs a program and prints every observation with the machine step at which it fired, plus the run's totals (`--json` for tooling). Step indices are pure functions of the source and inputs, so they are stable program points; the ruler is how a `prism exec suspend --at N` budget is picked deliberately, and suspend now reports where its cut fell on the observation timeline.
- Allocation witnesses: a failed `@ noalloc`, `fip`, or `fbip` certificate now explains itself, listing the first three concrete allocation reasons in evaluation order (a fresh constructor by name, a fresh tuple, a materialized closure, an uncertified or indirect call by callee, a primitive off the allocation-free list) with a trailing count for the rest. Witnesses are read off the reuse-lowered core, so a reported allocation is one the optimizer could not eliminate; accepted programs are byte-for-byte unaffected.
- Comprehensions: a guard-free comprehension `[ e for x in s ]` now lowers to a mapped and collected stream instead of a first-class effectful consumer thunk, so it fuses onto the state-threading tier: zero effect-operation cells and about two cells per element where the old path paid roughly two hundred, pinned by a new perf gate. Result list, element order, effect order, and single source evaluation are unchanged, and guarded or binding comprehensions keep the general pruning path byte-for-byte.
- Wire: `deriving (Stable)` now carries a compiler-computed shape digest as the class's one method (`shape_digest_of`, a per-type constant injected at the derive site from the same construction that seals frozen rungs), and `wire_encode_stable`/`wire_decode_stable` frame and open a value under its own digest with no hand-written digest string; a mismatched digest, wrong kind, truncation, or trailing bytes stays a hard failure. The explicit-digest helpers are renamed `wire_encode_value_with_digest`/`wire_decode_value_with_digest`, and a hand-written `Stable` instance is rejected, since the only method is compiler-computed and a manual instance could only forge a frozen contract.
- Wire/Bytes gates: the perf gate grows linear-slope pins on derived encode/decode, an in-place-extend pin on the buffer builder append path, a flat pin on the hex codec, and a static Core check that the derived container encoder threads the builder fold (allocation slope alone cannot catch a right-nested concatenation, since a buffer is one cell at any length). Parity coverage grows a full `buf_*` seam case (copy-on-shared writes, deterministic content `buf_hash`, the validating versus lossy byte-to-string split) and a hostile-frame case (wrong scheme, wrong kind, wrong digest, truncated body, hostile declared length, map-ordering violation), each failing through `Fail` rather than trapping.
- Deprecated the `string_of_bytes` builtin: use `bytes_to_string` (validating, returns an Option) or `string_of_buf` (explicit lossy repair) from `Data.Bytes`.
- `--time-compile` (or `PRISM_TIME_COMPILE=1`): one tab-separated timing row per compiler phase on stderr, carrying the phase name, wall time, abbreviated input artifact key, a cache-status column (always `cold` today; the field exists so the format survives a compile cache arriving), the output artifact key where one exists, and per-phase counts. Line-oriented on purpose, so scripts and tests diff shape rather than parse prose; program stdout is byte-identical with the flag on.
- Internals, all proven byte-identical by the frozen goldens and the stdlib root: the runtime-call and float builtin families collapse into one declarative registry (enum, surface names, hash tags, ABI, platform flags, type signatures, and wire order from one table per family, wire indices explicit with a compile-time density guard, so adding a builtin is one row instead of four synchronized tables); the def-store and continuation codecs share one byte substrate (varints, bounded lengths, blobs, tag numbering) while keeping their schemas separate; effect lowering's private Core child-mapper is deleted in favor of the one exhaustive traversal, so a new Core variant is a compile error at a single site instead of a silent catch-all; and the CLI binary shrinks from three thousand lines to a parse-and-dispatch shell over `src/cli/` modules, with package gates as a typed enum and the check-world report as typed structs rather than hand-built JSON.
- Soundness: a row variable introduced outside a row-polymorphic `forall` can no longer be solved to the rigid row that `forall` binds. The capture previously leaked silently (a solution outliving the binder that scoped it, the row analogue of an escaping skolem); it is now rejected with a pointed error naming the rigid row and the rule.
- Fixes: a data-type pattern synonym whose `view` clause is a bare expression now reports a pointed diagnostic instead of an internal compiler panic; native kont frame instrumentation is emitted only under its opt-in flag, shrinking default LLVM output while the metadata tables stay unconditional.

## 0.8.0

- Package world: package universes are now reported by digest and gated in CI. The first gate is deliberately typecheck-only.
- Web gallery: added the schedule-map resident and tightened same-origin teleport. Receivers now handshake by identity and code hash before resuming, while cross-origin mobility remains deferred to the typed mobility design.
- Instrumentation: source probes add named diagnostics whose bodies are skipped unless enabled, so disabled probes do not evaluate formatting work or perform side effects.
- Native mobility: native LLVM builds now carry continuation metadata, state-map rows, frame anchors, and optional entry-argument shadow frames. This identifies native frames by content-addressed code, without claiming arbitrary stack or register resume.
- Packages: source bundles are now loadable package artifacts. Published packages resolve through the signed package index, locked package and Std roots carry hash schemes, and path dependencies remain editable source.
- Packages: `packages/tzdb` is the first in-repo package seed, proving the publish/add/run path for data-as-package without freezing the real timezone database into Std.
- CLI: `prism run --examples` owns example discovery, stdin fixtures, retries, and aggregate failures. Ordinary `prism run file.pr -- args...` now forwards only program arguments.
- Surface cleanup: plain float comparisons use the numeric tower, negative integer literals work in polymorphic numeric code, and the deprecated dotted float operators warn through the normal deprecation path.
- Typechecker hardening: signature-written type variables stay rigid through renaming, closing a capture path between explicit `forall` binders and generated names.
- Verification tooling: Lean oracle fixtures are now compact hash manifests regenerated from source fixtures, eliminating the huge checked-in Core JSON diffs.
- Runtime plumbing: continuation code moved into its own runtime module, source probes gained their runtime gate, and the prelude has a message-carrying fatal hook.
- Compiler diagnostics: allocation checking now reports concrete witnesses for fresh constructors, uncertified callees, allocating primitives, indirect calls, and fresh closures.

## 0.7.0

- Mobility: suspend and resume a running program. `prism suspend --at N` pauses a run after N machine steps and writes the whole live continuation, its frame stack, its pending work, and every bound value, to a `kont` envelope; `prism resume` decodes it and runs to completion. The suspending run's output followed by the resuming run's output is byte-identical to one uninterrupted run. The envelope carries the program's namespace root as a bundle digest, so a continuation refuses to resume against code it was not captured in, and the same interpreter compiled to WebAssembly moves a running program between same-origin browser contexts, each re-verifying the bundle by hash.
- Types: a `Nat` kind for type-level dimensions. A parameter annotated `: Nat` ranges over compile-time natural numbers, so `Vec(a, n : Nat)` in `Data/Vec` carries an erased length index; dimensions unify by equality only, a length mismatch is a compile error naming both lengths, and there is deliberately no arithmetic on dimensions. An unannotated parameter still defaults to kind `Type`.
- Effects: named handler instances. `with f <- handler` installs a handler under a name and `f.op` dispatches to that instance, each desugaring to a fresh private effect so two instances of one effect never collide, with an escape check rejecting an instance that outlives its handler.
- Effects: per-operation grades. An effect operation is declared `fun`, `ctl`, or `final ctl`, and lowering-tier selection reads the grade per operation, so a multishot handler elsewhere in a program no longer drags an unrelated `var` loop off its fast tier.
- Effects: effect rows unify rather than subsume. Row equivalence is one unification relation over scoped labels with an open-tail variable and the old subsumption pass is deleted, so principal inference has a single rule and a handler-arm continuation row can no longer silently collapse to empty.
- Numerics: a real numeric tower with pinned, backend-invisible semantics. Integer literals take digit separators and scientific notation always denotes a `Float`, never colliding with unary minus; fixed-width `I64`/`U64` arithmetic wraps two's-complement, integer division truncates toward zero with the remainder taking the dividend's sign, and division or modulo by zero faults. Unary minus elaborates to a genuine `Comp::Neg` Core node rather than `0 - x`, so the `Float` lane is an IEEE sign flip preserving signed zero and every lane is byte-identical across the interpreter and both native backends. Negative literals on unsigned or non-numeric types are rejected at the caret.
- Numerics: floating-point transcendentals are owned rather than borrowed from the platform. Every elementary function routes through a `prism_m_*` wrapper over a vendored double-precision libm compiled `-ffp-contract=off`, and the native backend and the interpreter call the same object, so a transcendental is bit-identical on every backend by construction. The contract is determinism, not correct rounding.
- Numerics: `Data/Checked` layers the four Rust-style safe families over the raw operators through one `Checked` class, `checked_*` returning `None` exactly on overflow or a zero divisor, `saturating_*` clamping to the crossed lane bound, and `wrapping_*`/`overflowing_*` naming the wrap the raw operators already have.
- Incremental: effectful durable memos. `run_incr_durable_replay` records each memo's output beside its result and replays it on a warm hit, so a derivation that prints when it fires is warmed from the snapshot without running its thunk yet reproduces its output byte-for-byte, lifting the pure-up-to-`Fail` restriction the durable form required.
- Streams: a pull sequence and whole-program fusion. `Sequence` builds a pull sequence on an explicit `Step` co-structure with the full combinator vocabulary over a first-class value, complementing the prelude's single-source push streams for pipelines that advance two sources in step; a whole-program fusion pass collapses a recognized producer-consumer pipeline into one loop with no `Step` cells at `-O2`.
- IO: binary file IO. `read_bytes` is a `FileSystem` capability that reads a file as raw `Bytes` and is recorded for replay, its own operation rather than a lossy detour through `String`, and `write_bytes` is an `IO` output; `Data/Bytes` adds hex and base64 codecs alongside string conversions.
- Stdlib: new modules. `Json` is a total JSON codec with exact-or-error number decoding, `Time` is a virtual-clock date and duration module, `Cli` is an applicative command-line parser where one description yields the parse, the help, and the usage, and `Data/Ordered` supplies a branded ordering witness for statically coherent ordered containers. The typeclass tower gains `Data/Foldable` and `Data/Monad`.
- Deprecation: a `deprecated` annotation line preceding a declaration, recorded in the program and warned at each use site, with the fixed-width and float dot-operator builtins as its first consumers.
- Runtime: the monolithic C runtime is split into focused, independently warning-clean modules behind `prism_internal.h`, `prism_int`, `prism_float`, `prism_string`, `prism_array`, `prism_sort`, `prism_io`, `prism_mem`, and `prism_effect`, each held to the same `-Werror` and clang-tidy bar.
- Web: branching timelines. A simulation's full state is snapshotted, scrubbed to any frame by replay, and forked to continue a perturbed run as a pure function of the state and the step count, served beside the playground with a live content-address viewer.

## 0.6.0

- Deriving: structural `Serialize`, `Stable`, `Hash`, and `Arbitrary` instances, a rounded-out `Ord`, and `Identifiable` (an identity starter pack expanding to `Eq`, `Ord`, `Hash`, and `Show`). `Hash` falls out of the content-addressing fold rather than a hand-written instance.
- Serialization: a `stable` block, the release's one new language surface, freezes a type's versioned rungs with generated up/downgrade converters and a per-rung shape-digest golden, so editing a frozen rung is a compile error. It rides a pure, total wire codec (`lib/std/Wire.pr`) with a digest-checked envelope, an adjacent-version ladder, and a property-tested roundtrip. Opt-in; transport is a capability.
- Store: an on-disk two-layer content-addressed store (anonymous Core plus a name/type metadata layer) where equal hash implies a byte-identical artifact, gated by a from-scratch-versus-incremental oracle. Verification caching records a passed check against its hash, so an unchanged definition re-runs no tests, doctests, or parity checks and the cost of a change tracks its Merkle closure.
- Store: cross-program instance coherence, where a divergent canonical instance for a `(class, head)` already bound elsewhere is a hard error before any write. Content hashes are unified on the pre-optimizer core and canonicalized under renaming and reordering, so a definition's identity is independent of optimizer level and only a behavioral change moves a hash.
- Packages: a `prism.toml` dependency table (a local path, a `git` URL plus tag, or a bare hash pin) with a format-preserving manifest writer and a terminal `prism.lock`, resolved as a Merkle closure over a transport that re-hashes every fetched object, with a git-backed store transport. A signed package-identity-to-root index and an append-only transparency log make a published root attributable and a silent repoint detectable. New verbs: `prism add`, `why`, `export`, `publish`, and `audit`.
- Showcase: `prism diff` reports what changed between two versions by content hash plus its dependents cone; `prism run --record` and `prism replay` capture and re-run a `.replay` trace bit-for-bit; `prism debug` steps a trace forward and back as a reverse-step debugger; and `prism attest` compiles through two independent backends and attests their output is byte-identical.
- Attestation: a parity certificate is a content-addressed attestation that two backends agreed on a program. `prism attest` emits one once the parity oracle passes, `prism audit` verifies it against the signed index, and a failed certificate blocks the audit.
- Incremental computation: a new `Incr` effect and `run_incr` handler build a demand graph whose memo nodes cut off early, a recomputed value whose content hash is unchanged does not disturb its dependents, so a small edit re-runs only the affected cone. `run_incr_durable` persists the memo table for a warm restart whose output is byte-identical to a cold run (`examples/leaderboard.pr`).
- Concurrency: structured cancellation in `Concurrent`. `cancel` is a cooperative unwind that runs each fiber's finalizers exactly once, `on_cancel` installs a finalizer, `try_await` returns `Completed`/`Was_Cancelled` rather than hanging on a cancelled fiber, and a `scope` is fail-fast, cancelling siblings and re-raising a failed task at the scope boundary.
- Testing: `deriving (Arbitrary)` and a seeded `Quickcheck` harness, replayable through the `Random` handler and reporting a counterexample with its seed, plus an `--accept`/`--bless` mode that rewrites stale expected-output blocks in `.pr` doctests in place.
- Soundness: the skolem-escape hole is closed, so an outer existential can no longer be solved to a skolem bound under an inner `forall`; the offending call is rejected at its own caret.
- Core: match compilation uses join points, so N guarded arms emit Core linear in N rather than 2^N copies, and a committed tier manifest pins each corpus program to the effect-lowering tier it hits, turning a silently defeated fast path into a CI failure.
- Runtime: two memory-safety checks (a reuse token's cell fitting its payload, and mutable-cell validity) are now always-on rather than debug-only, and native float printing is an in-repo shortest-round-trip formatter, so interpreter parity holds by construction rather than by libc agreement. `Replay` durability is append-only and exactly-once, so a resumed run never re-prints work already done.
- Web: a determinism scrubber, a boids swarm you drag backward and forward on a timeline where every frame is a pure replay of one run to its step index, served beside the playground and REPL (`just scrub`).
- Verification: the reference grammar drift-checks its keyword set against the lexer, and the JSON differential bridge distinguishes `Int`/`I64`/`U64` widths, so a fixed-width overflow divergence reaches the Lean oracle.
- Stdlib: a `Blit` class for bulk in-place copy, with `String` and `Array` instances.
- CLI: help output condensed to a tighter, scannable summary.
- Fixes: guarded-arm native codegen, the re-enabled native `Ord` sort kernel, and a same-named-module import that resolved to the importing file itself on a case-insensitive filesystem.
- Tooling: a Nix flake for a reproducible dev shell and build.

## 0.5.0

- Types: row-kinded type parameters. A kind system (`Type`, `Row`, `Fun`) lets a type parameter range over effect rows (`type Cmd(a, e : Row)`); an unannotated parameter defaults to kind `Type`, so existing code is unchanged. With a Koka/Frank-style ambient-row discipline for effect operations, this makes effect-polymorphic handlers sound.
- Show: `show` is now a real typeclass method (`show : (a) -> String`) dispatched through the instance dictionary, not the type-directed compile-time generator it used to be. `deriving (Show)` produces the canonical structural instance: a record constructor prints with its field names (`T { f = v }`), a positional one prints `Name(v, ...)`, a nullary one its bare name, and a nested `String` renders quoted and escaped. Primitives (including `Unit`), `List`, `Option`, and `Result` carry instances, and a tuple gets a compiler-synthesized `Show` dictionary assembled from its components' instances. The `print`/`println` and string-interpolation renderer stays a separate total printer (an unresolved type falls back to the integer printer rather than demanding an instance) that inserts a top-level string raw, so `{x}` splices the string verbatim while `show(x)` quotes it; the derived instance and this printer are kept in lockstep by a consistency gate.
- Concurrency: cooperative async/await as a single handler, the new `Concurrent` stdlib module. An `Async` effect (`fork`/`yield`/`await`/`cancel`), buffered FIFO channels plus bounded channels with backpressure, and a structured `scope` nursery. Scheduling policy is itself a handler: `run_async` (FIFO) and `run_lifo` are two policies over the same effect, selected by `--scheduler cooperative|lifo` (or `PRISM_SCHEDULER`).
- Concurrency: a logical `Clock` capability (`now`/`sleep`) discharged by `run_clock`, threading a pure counter so time is virtual and runs are deterministic and replayable, a fifth capability beside `Console`/`FileSystem`/`Random`/`Env`.
- Concurrency: a deterministic cooperative schedule records and replays a concurrent run bit-for-bit (`examples/replay_concurrent.pr`). Capability-based sandboxing is documented (spec 7.8). `Preempt` is a reserved effect name for a future preemptive scheduler.
- Runtime: the whole-program CPS trampoline is now default-on, so a fiber that yields a million times runs in a 2 MB stack; a non-yielding fast path stays native. Opt out with `PRISM_TRAMPOLINE=0`.
- Verification: a Lean fixture runs the real scheduler core through the model's verified CEK machine, with `effect_tunnels` metatheory for a forked fiber's effects.
- Formatter: large declarations wrap canonically, one constructor/parameter/field per line and idempotently. `prism fmt` no longer descends into dotfile directories or `target`.
- Syntax: `without alloc`, a postfix signature suffix certifying a function and its whole call tree allocate no fresh heap cell. It carries the `fbip` check without `fip`'s linearity and bounded-stack requirements, composes with effect rows and `given` constraints, and rejects an offending allocation with a caret diagnostic. A terser `\ alloc` spelling (read as the result type with the `alloc` coeffect subtracted) is a checked-identical synonym, and the formatter now round-trips whichever spelling the author wrote instead of canonicalizing one to the other.
- Hashing: content-addressed Core. Every definition gets a behavior hash that commits to everything its compilation reads, so equal hash implies a byte-identical compiled artifact, plus structural shape digests for datatypes, effects, classes, and instances. The whole standard library folds to a single Merkle root, scheme-tagged `prism-core-hash-v1`. Exposed via `dump core-hash`, `dump shape`, and `dump stdlib-hash`; the stdlib docs carry the root as a fingerprint, and `tests/hash_parity.rs` gates the hash against the emitted LLVM IR.
- Hashing: a read-only inspection surface over the content-hash graph. `prism query` answers callers, transitive dependents, dependencies, and uses-of-type; `prism dump dupes` finds structural clones (definitions that hash identically); and `prism dump namespace` exports the two-layer store shape (the anonymous hashed layer plus a name/type metadata layer) in a versioned envelope. A per-type shape-digest golden gates format drift, generalizing the stdlib fingerprint to any type.
- Docs: a `prism docs` generator. Markdown API docs, one page per module, built from `-- |` doc comments and typechecker-inferred signatures, with rustdoc-style doctests (`--test` runs `prism` blocks in doc comments) and an mdbook preprocessor that live-checks code blocks. `--stdlib` renders the standard-library reference (the committed book Part III, CI-checked with `--check`).
- Web: an in-browser REPL, compiled to WebAssembly and served from the docs site (new `pages.yml` deploy).
- Fix: a dependency-graph scoping bug in type inference. The call-graph builder collected references by name without lexical scope, so a shadowed name could gain a spurious edge, merging a function into the wrong SCC and collapsing its inferred effect row. References are now collected scope-aware.
- Optimizer: `-O2` is now a distinct level rather than an alias of `-O1`. It runs a second inline/simplify iteration, so a call chain that only collapses to a single site after the first inlining round is still flattened. Both passes are idempotent, so the extra round is a no-op once a program reaches a fixed point.
- CLI: the environment knobs that tuned the optimizer and effect backends are now one canonical `DynFlags` set, read once at startup and threaded into the compiler instead of sampled deep inside individual passes. Each is also a CLI flag: `--no-native-effects`, `--no-trampoline`, `--core-lint`, `--opt-stats`, and `--dump-core SINK`, alongside the existing env vars.
- Internals: the experimental meta-continuation CEK effect backend is fenced behind a `cek-spike` cargo feature (off by default), so the path that is not yet validated against masking cannot be selected in a release build. The effect lowerer's two mutually-exclusive resume-reification modes are unified into a single enum, so a clause can no longer emit two encodings.

## 0.4.0

- Runtime: an optional structural backstop for the native runtime, built with `-DPRISM_RT_DEBUG`. Every cell dereference asserts a non-null, aligned, live heap pointer and every field read an in-bounds index, aborting with a diagnostic on a codegen or refcount bug instead of corrupting memory. It compiles out by default at zero overhead, needs no sanitizer toolchain, and a new CI step re-runs the parity corpus under it.
- Effects: the effect-lowering fast-path matchers now surface drift in release builds. A matcher that accepts a clause but finds its continuation-erasure post-condition violated still falls back to the correct non-fused lowering, but prints a one-time stderr warning, silenced by `PRISM_QUIET`, instead of degrading silently. Debug builds keep the loud assertion.
- Soundness: regression tests pin two properties. A `var`-using closure smuggled out through a call that returns it slips past the syntactic escape check, but its arrow row still carries the var's private effect, so principal inference rejects it as unhandled before the in-place lowering runs. And `Array` has copy-on-write value semantics, the property that makes the covariant treatment of `Array` element types sound.
- Hygiene: stop tracking stale codegen artifacts and ignore `*.bc`/`*.o`, so an in-tree build no longer leaves committable files.
- CLI: project builds now write the binary and its intermediates into a `target/` directory at the package root, rustc-style, instead of the current directory; `-o`/`--out` still wins and single-file builds are unchanged. A new `prism clean` removes that directory.
- Optimizer: the mid-level Core-to-Core tier is now a composable pass framework in the spirit of GHC's `CoreToDo`. One shared traversal replaces the hand-rolled Core walkers, and passes run from an ordered, level-keyed pipeline gated by a Core Lint well-formedness check, idempotence, and per-pass telemetry alongside the triple-backend parity oracle. The pipeline spans two stages around effect lowering, and the simplifier runs in the late stage on purpose, since running it before lowering degraded var/State fusion. A new `-O`/`--opt` flag selects the level: `-O0` is representation only, `-O1` (the default) adds dictionary specialization, a gentle simplifier, a bounded inliner, and scalar common-subexpression elimination, and `-O2` currently equals `-O1`. Compiled behavior at every level is byte-identical under the parity oracle.
- Optimizer: a new `--passes` flag drives the tier with an explicit ordered pass list, the LLVM `opt -passes=` / GHC `CoreToDo` analogue, overriding the `-O` level rather than augmenting it. The parser rejects an unknown or mis-staged pass name.
- Effects: the nondeterministic IO inputs are now real, handleable capability effects rather than untracked builtins. `read_int`/`read_line` perform `Console`, `read_file`/`file_exists` perform `FileSystem`, `rand` performs `Random`, and `getenv`/`args_count`/`arg` perform `Env`, so a function's row names which part of the world it reads, closing a soundness gap where these inputs were typed pure; the output file builtins now carry `! {IO}` too. The surface is unchanged: the inputs stay callable as prelude wrappers, discharged by a default tail-resumptive `run_io` world handler wrapped around `main` at zero cost. `Console`/`FileSystem`/`Random`/`Env` are now reserved effect names.
- Packaging: the macOS Apple Silicon binary now references the unversioned z3 and zstd dylibs rather than the exact versioned names baked in at build time, so it runs against whatever current z3 Homebrew provides; the 0.3.0 binary aborted at launch on a different z3 minor version.
- Effects: record and replay over the capability effects, a new `Replay` stdlib module. `record(action)` runs an action against the real world, logging every observation into an opaque trace and returning `(result, trace)`; `replay(trace, action)` re-runs it performing no real IO, discharging each operation from the trace. Replaying a recorded trace reproduces the original result.
- Effects: a `replayable` function annotation, orthogonal to `fip`/`fbip`, accepted only when the inferred effect row stays within `{Console, FileSystem, Random, Env, Exn, Fail}`; a row containing `IO` or any user-defined effect is rejected with a caret diagnostic. It certifies a function is reproducible from a recorded trace.
- Effects: durable record/replay in the `Replay` module. `serialize`/`deserialize` round-trip a trace through a total text format, and `durable(path, action)` is a crash-recoverable driver that replays the persisted prefix performing no real IO, then performs and persists each new observation, so a crash leaves a recoverable log. The flagship `examples/durable.pr` demonstrates crash and resume across two runs.
- Effects: console output is now an interceptable capability, so a durable resume no longer re-prints work it already did. `print`/`println` perform a new `Output` effect, discharged to the real printer by `run_io`, dropped by `replay`, and replayed-then-performed-once by `durable`. The routing is scoped to programs that `import Replay`; everywhere else `print`/`println` keep their `! {IO}` row, and `eprint`/`eprintln` and the file builtins stay `! {IO}`.
- Dependencies: `toml` 0.8 to 1.1, `wasm-bindgen` 0.2.125 to 0.2.126, `rustyline` 18.0.0 to 18.0.1, and `actions/checkout` 6 to 7.

## 0.3.0

- New surface syntax, every form sugar over the existing core:

| Sugar                           | Desugars to                                                                                    |
| ------------------------------- | ---------------------------------------------------------------------------------------------- |
| `x += e` (and `-=`, `*=`, `%=`) | `x := x <op> e`                                                                                |
| `while cond do <block>`         | `repeat_while(\() -> cond, \() -> <block>)`                                                    |
| `loop <block>` (may `break`)    | `repeat_while(\() -> true, \() -> <block>)`                                                    |
| `loop <block>` (no `break`)     | `forever(\() -> <block>)` (bottom-typed: never returns)                                        |
| `break` / `continue`            | `final ctl` perform of an internal `Break`/`Continue` effect, handled at the loop              |
| `return e`                      | `final ctl` perform of an internal `Return(a)` effect, handled at the function body            |
| `a ^ b`                         | `pow(a, b)`, the `Pow` class method (`Int` instance bignum-multiplies, `Float` is `pow_float`) |
| `a[i]`                          | `index(a, i)`, a failable read dispatched on `a`'s head type (`Elem ! {Fail}`)                 |
| `a[i] := v`                     | `a := index_set(a, i, v)`, the in-place (FBIP) setter, rebinding the `var`                     |
| `a[i] += e` (and `-=`, `*=`)    | `a[i] := a[i] + e` (the read makes it `! {Fail}`)                                              |
| `f(args, using d)`              | explicit instance selection (the old `f[d](args)`), `f`'s callee wrapped in the chosen dict    |

- Indexing `a[i]` / `a[i] := v` / `a[i] += e` over `Array` (Int key), `HashMap` (String key), read-only `String` byte, and `List`, dispatched at elaboration on the receiver's head type (no new class). A missing index performs `Fail` (`a[i] : Elem ! {Fail}`), so it composes with `??`/`?.`/`default`; writes are FBIP-in-place when uniquely owned, and nested `grid[i][j] := v` composes.
- Explicit instance selection moves from `f[name]` to `f(args, using name)`, freeing `[]` for indexing and converging override with first-class dictionary passing; `using` is reserved.
- Instance coherence: each `(class, type-head)` has one canonical instance that implicit resolution always selects, so ambiguity-at-use is gone. A lone instance is canonical by default; when several share a head one is named with a top-level `canonical Class(Head) = name`, and two undesignated instances are a coherence error at definition (caret plus designation hint). `f(args, using name)` stays the visible override, and a `newtype` is the way to a different default. No Core, runtime, or backend change, so the parity oracle is byte-identical. `canonical` is reserved.
- A right-associative power operator `^` (tighter than `*`), the method of a new `Pow` class: `2 ^ 10` is bignum-correct `Int`, `2.0 ^ 10.0` is `Float`, mixed `Int ^ Float` a type error. The prior integer `pow` is now `int_pow`.
- Imperative loops (`while`/`loop`) lower to a tail-recursive prelude driver (constant stack, no per-iteration allocation), an unconditional `loop` to the bottom-typed `forever`. `break`/`continue`/`return` compile to non-resumable performs of internal, fully-handled effects, so none surfaces in a function's row, and a loop installs a handler only for the keyword it uses. The prelude's old `while` is now `repeat_while`.
- Principal effect-row inference: a lambda delimits its effects onto its own arrow row, the arrow is covariant in that row (a pure function fits any effectful context via row subsumption), and the call-graph set pass is dropped as a row seed so the inferred row is the single source of truth. Builtins carry their effect row on the type, so inference attributes `IO`/`Exn`/`Fail` directly; rows display in canonical name-sorted order; and definitions are inferred in dependency-SCC order so a forward reference sees a generalized type.
- `mask` over the sole handler of an effect now leaves the operation genuinely unhandled (the label stays in the row) instead of inferring it pure and hitting an effect-reconciliation ICE at lowering.
- Two source warnings (the prelude is exempt): an unused local binding and a name shadowing one in scope (a leading `_` and a consuming rebind `let s = f(s)` are exempt). Annotations are name-checked uniformly across parameters, returns, constraints, and rows: an undeclared effect or constructor is a hard error, and an annotation broader than the inferred row warns.
- Standard library split into on-demand modules under `lib/std` (`Data.Char`/`List`/`Map`/`Maybe`/`Result`/`Set`/`String`), shrinking the always-loaded prelude; `Set` gains `set_union`/`set_intersection`/`set_difference`, and a project may replace the built-in prelude via `[package] prelude` in `prism.toml`.
- CLI: a bare `prism program.pr` compiles a single file to a native binary named after the source (`-o` overrides, `--mlir` selects the backend); `prism build` is the project verb (nearest `prism.toml`) and interpreting is explicit via `prism run`.
- Local monadification confines the free-monad form to the component entangled with an escaping effect, keeping unrelated functions on their fused evidence/state path and falling back to whole-program lowering only when the split is unclean. The fallback now warns on stderr by default (silence with `PRISM_QUIET`), naming the functions that lost fusion.
- Constant-stack effect handlers: a closed free-monad handler is driven by one self-tail-recursive `{n}@region` loop, so a tail-resumptive or function-answer state handler resumes via `musttail` in constant native stack (a `perf_gate` `State` loop goes from ~10^5 to millions of iterations in a 2 MB stack). On by default (opt out with `PRISM_NATIVE_EFFECTS=0`); it changes the driver, not the one-`EOp`-cell-per-operation reification.
- Compiler-internal correctness, with no change to accepted programs or output: existential scope is enforced unconditionally at `solve`; numeric defaulting is deferred to one pass at generalization; and FBIP reuse becomes a scoped `WithReuse`/`Reuse` core node, making free-once and spend-at-one-allocation well-formedness properties of the term.
- The Lean core model (`models/`) grows from a substitution small-step into a mechanized CEK machine proved to implement a big-step semantics, with unique normal forms, a progress trichotomy, and effect progress/unhandled lemmas, all `sorry`-free over `propext`/`Quot.sound`. It runs as a third differential oracle via a new `prism dump core-json` phase, checked against the interpreter across a fixture corpus.
- Hardening: the bignum allocator is overflow-checked like every cell, each builtin's calling convention is derived from an exhaustive `Builtin` method (so a new builtin cannot ship without declaring its ABI), and the reference grammar is single-sourced in `models/grammar.ebnf`, sliced into the spec by mdbook anchors.

## 0.2.0

- Fixed-width bitwise and shift builtins on the I64/U64 lanes: `i64_and`/`i64_or`/`i64_xor`/`i64_shl`/`i64_shr` and their `u64_*` counterparts. and/or/xor share one bit pattern across lanes; `i64_shr` is arithmetic, `u64_shr` logical; shift counts are taken modulo 64.
- `system(String) -> Int` runs a shell command and returns its exit code, and `eprint`/`eprintln` write to stderr, so a program can drive external tools and emit diagnostics off the stdout stream.
- Superclasses: a class may declare another as a superclass (`class Ord(a) given Eq(a)`). Each instance carries a resolved superclass dictionary as a leading field of its dict cell, and a `given Ord(a)` constraint discharges an `Eq(a)` obligation by projecting it, found automatically from the instances in scope. The prelude's `Ord` now requires `Eq`.
- Growable mutable `Array(a)` (`array_new`/`array_empty`/`array_len`/`array_get`/`array_set`/`array_push`), an ordinary reference-counted heap cell so drops recurse into its elements. `array_set` and `array_push` write in place when the array is uniquely owned (FBIP) and copy otherwise; `array_push` doubles capacity when full, so appends are amortized O(1). The prelude adds `array_of_list`.
- `string_of_array` and the prelude `concat_all`/`array_of_list` build a string from many chunks in a single allocation, replacing the quadratic right-nested `concat` chain.
- Prelude `HashMap(v)`: a separate-chaining hash table with String keys built on the growable array (`hm_new`/`hm_insert`/`hm_lookup`/`hm_member`/`hm_get_or`/`hm_keys`/`hm_values`/`hm_size`/`hm_to_list`/`hm_delete`/`hm_from_list`/`hm_adjust`), doubling its bucket count past load factor 1. Keys hash by a fixed-width FNV-1a written in the language, so iteration order is a deterministic function of the inserts.
- O(1) byte access: `byte_at`/`byte_len` (UTF-8 unaware) and `string_of_bytes`, so a lexer or hash scans raw bytes in linear time. `array_pop` rounds out the array API, and `array_foldl`/`array_to_list` are added to the prelude.
- Surface fixed-width arithmetic: `i64_*`/`u64_*` `add`/`sub`/`mul`/`div`/`rem`/`cmp` (wrapping, no bignum promotion), enabling a real fixed-width hash in userland.
- Higher-kinded types: a class parameter may range over a type constructor (kind `* -> *`), applied as `f(a)` in method signatures, with instance resolution keyed on the head constructor. The prelude adds the `Functor`/`Applicative`/`Monad`/`Foldable`/`Traversable` tower with `List` and `Option` instances. `fmap`/`traverse` are effect-polymorphic, so the per-element effect row threads through instead of an `Applicative` wrapper (effects, not do-notation).
- String-utility prelude: character classifiers (`is_digit`/`is_alpha`/`is_alnum`/`is_space`/`is_upper`/`is_lower`, `to_upper_c`/`to_lower_c`), `starts_with`/`ends_with`/`contains`/`index_of`, `to_upper`/`to_lower`/`trim`, and `args()`.
- `prism fmt` separates top-level declarations with a blank line.
- Editor tooling: a dependency-free Neovim highlighter under `scripts/nvim/` (an `ftdetect/` filetype map for `*.pr` plus a `syntax/` highlighter), with its keyword set mirrored from `src/lex/token.rs` so it tracks the lexer.
- A project and module system. A `prism.toml` manifest (`[package] name`, `[bin] entry`) plus a `src/` tree make a multi-file project that `prism run`/`build` compiles, resolving module paths from the project root rather than the entry file's directory (single-file invocation is unchanged). Name resolution now canonicalizes every top-level definition to a module-qualified symbol (`Data.Map.insert` for exports, `Data.Map@helper` for privates), so qualified references (`M.x`) reach disjoint namespaces and two modules may export the same short name and coexist; selective imports (`import M (a, b)`) bring only the listed names into bare scope, and `pub` controls what an importer can reach. Instances record their defining module: an orphan instance (defined apart from both its class and its type) and instances overlapping across modules are reported as warnings, and an ambiguity names each candidate's module.
- Module-system follow-ups: `pub import M (x)` re-exports the named items through the importing module, with chains resolving transitively to the original definition; a module's full dotted path qualifies a reference (`Geo.Util.one`), not only its last component (`Util.one`); and orphan/overlapping-instance warnings render with a source caret when they point into the program being compiled.
- Static `fip`/`fbip` checking of the FP^2 discipline. `fbip` proves zero fresh allocation and that an annotated body calls only annotated, allocation-free functions; `fip` additionally proves linearity (each owned non-immediate binding consumed at most once, checked on the source term with scalars exempt) and bounded stack (every recursive call in the call-graph SCC is a tail call or a single tail-modulo-cons/-add). The tail/TRMC classification is shared with codegen (`src/core/tailrec.rs`), so an accepted `fip` function always lowers to a loop; `fip` may call only `fip`, `fbip` may call either. Recursive accumulator/TRMC functions like `rev_onto` and `bump` now type-check as `fip` and run in constant heap and stack.
- A cyclic superclass hierarchy (`class A given B`, `class B given A`) is now reported as a `superclass cycle: A -> B -> A` error at class-construction time, instead of overflowing the stack the first time the chain is walked at a use site.
- Pattern coverage now treats an irrefutable `if true` guard as always-matching: such an arm completes exhaustiveness like an unguarded one (no spurious non-exhaustive error) and shadows any later arm (which is reported unreachable). Fallible guards keep their conservative non-covering treatment.
- The `fip` bounded-stack rejection now explains when a function joins the tail-recursion group only because another function flows as a first-class value (a capture) rather than through a direct call cycle, pointing at the fix (call directly or annotate `fbip`). The set of accepted programs is unchanged; only the diagnostic is sharper.
- Rank-N polymorphism is predicative: a `forall` written directly as a type-constructor argument (`List(forall a. (a) -> a)`) is now rejected at the annotation with a clear "impredicative type" message naming the constructor, instead of surfacing later as a confusing `expected a, got Int` mismatch from a leaked rigid variable. Higher-rank types remain available as function parameters, results, and declared data fields (a polymorphic field carries a `forall` through a generic container).

## 0.1.0

Initial release.

- Strict, impure functional language with ML-family surface syntax: ADTs, pattern matching, parametric polymorphism, and a prelude of `Option`/`Result`/`List`/`Map` combinators.
- Hindley-Milner type inference with bidirectional, higher-rank (rank-N) checking and subsumption.
- Type classes by dictionary passing, with named instances and `deriving (Eq, Ord, Show)`.
- Algebraic effects and handlers: inferred, extensible effect rows via row polymorphism; multishot `resume`; `final ctl` non-resumable clauses; scoped/masked handlers and forwarding.
- Evidence-passing compilation of handlers, with tail-resumptive clauses lowered to direct calls and a free-monad fallback when effects escape tracking.
- Exhaustiveness and redundancy checking, plus decision-tree pattern-match compilation.
- First-class optics: record-update paths, view patterns, bidirectional pattern synonyms, and `deriving (Lens)`.
- Stream fusion: effectful producer/transformer/consumer pipelines fuse to zero intermediate allocations.
- Verse-inspired failure model (`fail`, `guard`, `??`, `?.`, `transact`) and structured error handling, both built on effect handlers.
- Deterministic memory management via Perceus reference counting with in-place reuse (FBIP), no garbage collector.
- Call-by-push-value core in A-normal form with tail-call optimization and tail recursion modulo cons.
- Three backends kept byte-identical: a tree-walking interpreter, native code via LLVM, and a text-MLIR backend.
- Lean model of the core with a machine-checked determinism theorem.
- Compiles to WebAssembly, so the language runs in the browser.
- Tooling: interactive shell, `run`, `build`, `check`, `fmt`, and phase `dump`.
